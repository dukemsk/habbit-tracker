# Заметки для будущей версии

## Текущие решения

- Быстрые кнопки `сделал сегодня` и `не сделал сегодня` убраны из карточки. Пользователь отмечает конкретный день прямо в сетке дат.
- Кнопка удаления находится в верхней части карточки рядом с названием: в покое это маленький серый кружок, при наведении появляется красный фон и крестик.
- Форма добавления привычки на широком экране показывается как левая панель, на узком экране открывается как модальное окно.
- Срок привычки редактируется из строки дат в карточке через компактный popover.
- Перетаскивание привычки начинается только с ручки с тремя полосками, чтобы не конфликтовать с датами и кнопкой удаления.
- Фон названия показывает доминирующий статус: мягкий зелёный для большего числа выполнений, мягкий красный для большего числа пропусков.
- Статистика в карточке показывает только `Сделано` и `Не сделано`.
- `Без отметки` не выводится в карточке, но остаётся в бизнес-логике как вычисляемое значение.
- Если невыполненных дней больше, `Не сделано` показывается первым. Если выполненных больше или значения равны, первым показывается `Сделано`.
- Экспорт открывается отдельным HTML-шаблоном в новом окне и рассчитан на печать или сохранение через системный PDF-диалог.
- Дни в экспортном шаблоне остаются пустыми: это заготовка для ручного заполнения, а не копия текущих отметок.
- Старый формат данных с массивом `completed` мигрирует в новый формат `statuses`.

## Что важно не сломать

- Период считается включительно.
- Нельзя сокращать активную привычку раньше сегодняшнего дня или отрезать дни с отметками.
- Повторное нажатие на дату должно позволять исправить ошибку.
- Красный статус не должен увеличивать прогресс.
- Удаление привычки должно требовать подтверждения.
- Порядок привычек должен сохраняться после перетаскивания и обновления страницы.
- Экспорт должен сохранять текущий порядок привычек и количество кружков по `period`.
- Приложение должно продолжать работать как статическая страница без сборки.

## Возможные следующие улучшения

- Добавить редактирование названия, цвета и диапазона уже созданной привычки.
- Добавить фильтры: активные, завершённые, с пропусками.
- Добавить импорт данных.
- Добавить недельный или месячный режим просмотра.
- Добавить браузерные e2e-тесты.

## Рабочее правило для следующих изменений

Если меняется правило поведения, сначала обновить `habitLogic.js` и тесты, затем интерфейс. Если меняется только внешний вид, достаточно обновить `styles.css` и проверить страницу вручную.
