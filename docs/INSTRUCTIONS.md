# Инструкция по проекту

## Что это

Это статическое веб-приложение трекера привычек на JavaScript. Оно работает без сервера: пользователь открывает `index.html`, добавляет привычки, выбирает период и отмечает каждый день как выполненный или невыполненный.

## Основные файлы

- `index.html` - структура интерфейса, модальные окна и шаблон карточки привычки.
- `styles.css` - визуальное оформление, адаптивность, цвета статусов и модальные окна.
- `habitLogic.js` - чистая бизнес-логика: даты, периоды, статусы, подсчёты, миграция старых данных.
- `app.js` - связь интерфейса с бизнес-логикой, работа с DOM и `localStorage`.
- `tests/habitLogic.test.js` - тесты для чистых функций из `habitLogic.js`.

## Как запускать

Открыть `index.html` в браузере. Для локальной проверки можно запустить простой сервер из папки проекта:

```bash
python3 -m http.server 4174
```

После этого открыть `http://127.0.0.1:4174/`.

## Как запускать тесты

```bash
/Users/duke/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node tests/habitLogic.test.js
```

Если в системе установлен обычный Node.js, можно использовать:

```bash
node tests/habitLogic.test.js
```

## Шаги разработки новой версии

1. Сначала менять или добавлять правила в `habitLogic.js`.
2. Добавлять тесты в `tests/habitLogic.test.js`.
3. Обновлять `app.js`, если изменилось поведение интерфейса.
4. Обновлять `styles.css`, если изменился вид карточек, модальных окон или статусов.
5. Проверять сценарии вручную: добавление привычки, заданный период, отметки дат, удаление с подтверждением.
6. Дополнять этот файл и `docs/DEVELOPER_NOTES.md`, если появилось важное решение для будущих версий.
