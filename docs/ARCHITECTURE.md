# Архитектура

## Общая схема

```text
index.html
  -> подключает habitLogic.js
  -> подключает app.js

habitLogic.js
  -> чистые функции
  -> тестируется отдельно

app.js
  -> читает DOM
  -> вызывает habitLogic.js
  -> сохраняет данные в localStorage

styles.css
  -> отвечает за внешний вид
```

## Модуль `habitLogic.js`

Это ядро приложения. В нём нет прямой работы с DOM и браузерными элементами интерфейса.

Отвечает за:

- преобразование дат;
- расчёт периода;
- построение массива дней;
- проверку границ периода;
- переключение статусов;
- подсчёт выполненных и невыполненных дней;
- сортировку статистики;
- определение доминирующего статуса карточки;
- обновление срока привычки с валидацией;
- перестановку привычек в массиве;
- подготовку строк для печатного экспорта;
- миграцию старого формата данных.

Такой модуль можно тестировать отдельно и переиспользовать, если позже появится backend, сборщик или мобильная версия.

## Модуль `app.js`

Это слой интерфейса.

Отвечает за:

- открытие и закрытие модальных окон;
- добавление привычки из формы;
- рендер карточек;
- обработку кликов по датам;
- редактирование срока через popover в карточке;
- перетаскивание карточек за drag-handle;
- открытие печатного шаблона экспорта;
- подтверждение удаления;
- сохранение и загрузку из `localStorage`.

`app.js` не должен заново изобретать бизнес-правила. Если правило можно проверить тестом, оно должно жить в `habitLogic.js`.

## Файл `index.html`

Содержит:

- шапку приложения;
- кнопку добавления привычки;
- список привычек;
- форму создания привычки;
- диалог подтверждения удаления;
- диалог настройки экспорта;
- шаблон карточки привычки.

Карточка привычки создаётся из `<template id="habitTemplate">`.

Форма создания привычки использует один и тот же `<dialog>`:

- на широком экране она открыта как левая панель;
- на узком экране она работает как всплывающее окно.

## Файл `styles.css`

Содержит:

- базовую тему;
- сетку страницы;
- стили карточек;
- стили модальных окон;
- состояния дней: зелёный `done`, красный `missed`;
- статусный фон карточки и названия;
- popover редактирования срока;
- адаптивность для мобильного экрана.

## Поток данных

1. Пользователь заполняет форму в диалоге.
2. `app.js` собирает данные формы.
3. `habitLogic.js` рассчитывает период и нормализует правила дат.
4. `app.js` добавляет привычку в массив `habits`.
5. Массив сохраняется в `localStorage`.
6. Интерфейс перерисовывается.

Пользовательская сортировка хранится порядком элементов в массиве `habits`. Отдельное поле `priority` не добавляется.

Редактирование срока обновляет `startDate`, `endDate` и `period` существующей привычки. Отметки внутри нового диапазона сохраняются.

Экспорт берёт строки через чистую функцию `getExportRows(habits)`, затем `app.js` открывает отдельный HTML-документ в новом окне. В этом документе нет доступа к `localStorage`: он получает только готовые строки `name`, `period` и `color`, а печать запускается через кнопку `Печать / PDF`.

## Тестовая стратегия

Тесты покрывают чистую бизнес-логику:

- даты;
- периоды;
- статусы;
- подсчёты;
- сортировку статистики;
- доминирующий статус;
- обновление срока;
- перестановку привычек;
- подготовку строк для экспорта;
- миграцию старых данных.

DOM-поведение пока проверяется вручную в браузере. Если проект вырастет, следующий шаг - добавить браузерные end-to-end тесты для сценариев пользователя.
