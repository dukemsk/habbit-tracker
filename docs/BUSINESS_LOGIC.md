# Бизнес-логика

## Проблема

Пользователь хочет внедрять новые привычки, но без фиксации результата легко потерять ритм. Нужен простой способ видеть, какие дни были выполнены, какие пропущены и как меняется прогресс за выбранный период.

## Для чего нужно приложение

Трекер помогает:

- добавить привычку с понятным сроком;
- выбрать стандартный период или свой диапазон дат;
- изменить срок уже созданной привычки без удаления;
- расставить привычки по личному приоритету;
- каждый день отмечать результат;
- быстро увидеть соотношение выполненных и невыполненных дней;
- исправить случайную отметку повторным нажатием на дату.
- экспортировать текущий список привычек в печатный шаблон.

## Сущности

### Привычка

Привычка хранит:

- `id` - уникальный идентификатор;
- `name` - название привычки;
- `startDate` - дата начала;
- `endDate` - дата окончания;
- `period` - количество дней, включая начало и конец;
- `color` - цвет карточки;
- `statuses` - объект отметок по датам.

### Статус дня

День может быть в одном из трёх состояний:

- без отметки;
- `done` - сделано, отображается зелёным;
- `missed` - не сделано, отображается красным.

При нажатии на дату статус меняется по кругу:

```text
без отметки -> сделано -> не сделано -> без отметки
```

## Правила периода

Период считается включительно: если дата начала `2026-06-19`, а дата окончания `2026-06-21`, период равен 3 дням.

Стандартные периоды:

- неделя - 7 дней;
- месяц - 30 дней;
- 3 месяца - 90 дней;
- год - 365 дней.

Заданный период позволяет выбрать любую дату начала и окончания.

Срок уже созданной привычки можно изменить из карточки. Нельзя сохранить диапазон, где дата окончания раньше даты старта, нельзя сократить активную привычку раньше сегодняшнего дня и нельзя отрезать дни, где уже есть отметки.

## Правила прогресса

Основной прогресс считается только по зелёным дням:

```text
сделанные дни / все дни периода
```

Красные дни не увеличивают прогресс, но показываются в статистике.

Статистика показывает только два значения:

- `Сделано`;
- `Не сделано`.

Значение с большим количеством показывается первым. `Без отметки` не выводится в интерфейсе карточки, чтобы не перегружать экран.

Если выполненных дней больше, название получает мягкий зелёный фон. Если невыполненных больше, название получает мягкий красный фон. При равенстве фон названия остаётся нейтральным.

## Удаление

Удаление привычки всегда проходит через подтверждение. При удалении стирается сама привычка и все её отметки.

## Приоритет

Порядок привычек в списке считается пользовательским приоритетом. Пользователь может перетащить привычку за ручку с тремя полосками и поставить её выше или ниже.

## Экспорт

Экспорт создаёт отдельный печатный шаблон для ручного заполнения на бумаге или в PDF.

Правила экспорта:

- в шаблон попадают все текущие привычки;
- порядок строк совпадает с текущим пользовательским приоритетом;
- заголовок задаётся пользователем, по умолчанию используется `План привычек`;
- слева показывается название привычки в мягком цветном блоке;
- справа показываются пустые кружки дней от `1` до `period`;
- зелёные и красные отметки из приложения в шаблон не переносятся.

Экспорт не сохраняет файл автоматически. Пользователь открывает шаблон и использует системную печать браузера, чтобы распечатать или сохранить PDF.

## Хранение данных

Данные сохраняются в `localStorage` браузера под ключом `habit-tracker-items`. Это подходит для первой версии без аккаунтов и сервера.
